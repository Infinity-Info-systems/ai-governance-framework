# AI Use Case Review Map

## Purpose

The review map connects AI use case types to the controls and review depth they require.
It helps teams avoid over-reviewing low-risk use cases and under-reviewing high-impact ones.

## Use Case Types

- internal productivity assistant
- customer-facing chatbot
- forecasting model
- decision support model
- automated action model

## Review Considerations

- data sensitivity
- user impact
- explainability
- bias risk
- approval requirement
- automation level
- brand and trust impact

## Example Map

| Use Case | Review Depth | Main Risks |
| --- | --- | --- |
| Internal assistant | Standard | Data leakage, inappropriate output |
| Customer chatbot | Enhanced | Safety, privacy, brand risk |
| Decision support | Enhanced | Explainability, bias, accountability |
| Automated action | Full board review | Operational and business risk |

## Use

This map should help teams decide how much governance effort a proposed AI use case needs before release.

## Outcome

Review work becomes easier to manage when every stakeholder can see the same path and the same exit criteria.
