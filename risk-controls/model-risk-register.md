# Model Risk Register

## Purpose

Track model risks, owners, and remediation actions for AI use cases.

## Suggested Fields

- model name
- risk description
- owner
- impact
- mitigation
- review date
- status
- evidence link

## Example Risks

- poor explainability
- training data issues
- drift
- unsupported automation

## Register Guidance

Use the register to keep active model risks visible until they are mitigated, accepted, or retired.
