# Cloud AI Governance

## Purpose

Cloud AI governance ensures AI services deployed in cloud environments follow consistent controls, access standards, and risk reviews.

## Core Areas

- platform access
- model deployment controls
- logging and monitoring
- data handling
- vendor and service review
- release approvals
- environment segregation

## Evidence To Collect

- deployment controls
- access review records
- logging configuration
- service approvals
- model release notes

## Use

Use this page when an AI capability is being delivered through a cloud platform and needs both cloud controls and AI-specific governance.
