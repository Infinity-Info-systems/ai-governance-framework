# Data Security And Privacy

## Purpose

This doc defines how AI use cases should manage sensitive data, access, retention, and privacy obligations.

## Core Areas

- data classification
- access control
- retention
- masking or anonymization
- privacy review
- source validation
- data minimization

## Evidence To Collect

- data inventory
- privacy review
- access controls
- retention rules

## Practical Use

This page should help reviewers confirm that the AI use case is using the minimum necessary data and handling it in a controlled way.
